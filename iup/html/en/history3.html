History of Changes of Version 3.x

Version 3.0  ALPHA (CVS module "iup3" - 14/Oct/2008)

General

  • Changed: ATENTION - Now attributes are stored in the hash table only if not processed or choosed by the element class implementation.
  • Changed: SAVEUNDER dialog attribute now is also available in Motif.
  • Changed: IupControlsClose function does nothing, declaration still remains for compatibility.
  • Changed: IupGetAttribute, IupSetAttribute and IupStoreAttribute can also be used to access global attributes using NULL as element.
  • Changed: TIP and ZORDER attributes are now non inheritable.
  • Changed: Removed the conflicts: K_BS=K_cH, K_TAB=K_cI and K_CR=K_cM.
  • Changed: The global attributes TRUECOLORCANVAS, SYSTEMLANGUAGE, SHIFTKEY and CONTROLKEY are now available in all drivers.
  • Changed: DROPFILES_CB callback is now available for all controls. It is only activated using DRAGDROP attribute. It is active by default only for IupCanvas and IupDialog.
  • Changed: the default value of the IupDialog Windows attribute COMPOSITED is now YES.
  • Changed: MAP_CB is now available for all controls.
  • Changed: attribute FONT now uses a common a more flexible definition for all drivers.
  • Changed: child dialogs were automatically destroyed by the system but not by IUP. IupDestroy is now automatically called for child dialogs when the parent is destroyed.
  • Changed: IupAppend and IupDetach can now be used for dynamic creation of menus, IupSbox, IupCbox and IupTabs, even after the element is mapped.
  • Changed: IupDetach will now automatically unmap the element.
  • Changed: IupAppend will now return the actual parent.
  • Changed: ALIGNMENT attribute in IupLabel now includes vertical alignment values.
  • Changed: the default FONT in Motif is now "Fixed, 10".
  • Changed: IupImageLibOpen will now only register names, but will not load the images. New 32bpp images for Windows. GTK aliases are also available. Many new images. IupImageLibClose removed, loaded images will now be automatically unloaded.
  • Changed: in the IupCanvas if ACTION is defined nothing is painted in the canvas now also in Motif. BORDER is now also supported in Motif. Now scroolbar parameters min, max, page size and line size are updated when DX/DY are updated. POSX and POSY will only update the position of the scrollbar. Automatic hide of the scrollbar now works also in Motif.
  • Changed: LANGUAGE global attribute default from PORTUGUESE to ENGLISH.
  • Changed: the NO_COLOR attribute is deprecated, now it simply sets the BGCOLOR attribute in IupCells.
  • Changed: in IupColorBrowser moved from HLS to HSI, added support for resize, anti-aliasing, support for BGCOLOR attribute, feedback for ACTIVE attribute, feedback for focus. New HSI attribute. New support for mouse wheel to change Hue. New support for PgDn and PgUp keys to change Hue.
  • Changed: IupFrames now are native parents of their children.
  • Changed: IupButton now supports text with more than one line.
  • Changed: IupUpdate now only mark the control to be redraw instead of redrawing at the function call.
  • Changed: renamed MARGIN attribute to PADDING in IupGauge.
  • Changed: IupMultiline is now implemented as IupText with MULTILINE=YES.
  • Changed: the ACTION callback in IupText now does NOT process navigation keys. It is called only if the text is edited. The callback now is called before the text is updated on screen.
  • Changed: the SELECTION and CARET attribute in Windows of IupText do NOT change the focus anymore. The NC attribute now only restricts keyboard input.
  • Changed: IupVal implemented as a native control. Attributes HANDLER_IMAGE and HANDLER_IMAGE_INACTIVE are not supported anymore in IupVal.
  • Changed: IupCbox is not based on IupCanvas anymore.
  • Changed: now IupText also supports the DROPFILES_CB, BUTTON_CB and MOTION_CB callbacks. CARET_CB now includes 0 based character position.
  • Changed: default value for ALIGNMENT in IupZbox is now "NW".
  • Changed: IupTabs implemented as a native control. Attributes ALIGNMENT, FONT_ACTIVE, FONT_INACTIVE, TABSIZE and REPAINT are not supported anymore in IupTabs.
  • Changed: all dialogs and all elements that have names are now automatically destroyed in IupClose.
  • Changed: An IupGLCanvas when inside an IupFrame in Win32 will now work normally. But the dialog COMPOSITE attribute must be NO.
  • Changed: The BGCOLOR is now ignored in IupLabel, IupFrame, IupToggle (for the text) and IupVal. They will use the background color of the native parent.
  • Changed: The HIGHLIGHT_CB, OPEN_CB and MENUCLOSE_CB callbacks now work normally for popup menus. HIGHLIGHT_CB is called for items and submenus. OPEN_CB and MENUCLOSE_CB are called only for menus. TITLE for submenus can now be changed after the element is mapped. Children can be added or removed from menus even after the menu is mapped.
  • New: GTK driver, available in UNIX and Windows.
  • New: global attributes APPSHELL, XDISPLAY, XSCREEN, XSERVERVENDOR, XVENDORRELEASE in Motif.
  • New: MODKEYSTATE global attribute in all drivers.
  • New: key definitions: K_acute, K_ccedilla, K_Print, K_Menu.
  • New: key definitions for the system key modifier K_y*. In Windows this is the Windows key and in Mac this is the Apple key.
  • New: "UPARROW" cursor in Motif. New cursors "RESIZE_NS"  and "RESIZE_WE". Updated cursor documentation with pictures of all pre-defined cursors.
  • New: dialog attributes: MINSIZE and MAXSIZE. InWindows MINSIZE is ignored for systems with multiple monitors. The Windowing system may impose a minimum default limit for the dialog that includes the title bar with all it buttons.
  • New: common attribute CLIENTSIZE. Returns the size of containers excluding their decoration.
  • New: ENTERWINDOW_CB, LEAVEWINDOW_CB for all controls.
  • New: DROPFILES_CB, RESIZE_CB callbacks for the IupDialog.
  • New: IUP_CURRENT and IUP_CENTERPARENT positions for IupShowXY and IupPopup.
  • New: IUP_HIDE and IUP_MAXIMIZE flags for SHOW_CB callback.
  • New: MODAL attribute for the IupDialog.
  • New: function IupGetClassType.
  • New: IupGetIntInt utility function.
  • New: IUP_ASSERT compile flag.
  • New: internal documentation and Guide to create new controls. Now all the controls use the same architecture using the same base class.
  • New: internal code reorganization. More clear and simple to create controls and drivers. All comments are now in English.
  • New: TIP additional attributes: TIPFONT, TIPDELAY, TIPBGCOLOR, TIPFGCOLOR, TIPBALLON (Windows Only), TIPBALLONTITLE (Windows Only), TIPBALLONTITLEICON (Windows Only), TIPVISIBLE. Not available in GTK.
  • New: IupColorDlg, IupFontDlg and IupMessageDlg native pre-defined dialog as elements.
  • New: support for 24 and 32 bpp images using IupImageRGB and IupImageRGBA constructors.
  • New: attributes PADDING, ELLIPSIS, WORDWRAP and MARKUP for IupLabel.
  • New: functions IupUnmap and IupReparent.
  • New: LINEX, LINEY, XAUTOHIDE and YAUTOHIDE attributes for the IupCanvas scrollbar.
  • New: SHOWHIDDEN attribute for IupFileDLg. Preview canvas support for the Motif driver.
  • New: functions IupInsert and IupGetClassAttributes.
  • New: function IupUpdateChildren.
  • New: CHARSIZE common attribute.
  • New: font face name mappings for Courier, Times and Helvetica.
  • New: FLOATING attribute to control the inclusion of the element in layout processing for IupHbox, IupVbox and IupZbox.
  • New: HOMOGENEOUS attribute to control the spacing in layout processing for IupHbox and IupVbox.
  • New: EXPANDCHILDREN attribute to control the expansion in layout processing for IupHbox and IupVbox.
  • New: IUP_IGNORE return code accepted for IDLE_ACTION callback to automatically remove the callback.
  • New: PADDING, MARKUP, FOCUSONCLICK and ALIGNMENT attributes for IupButton.
  • New: common callback UNMAP_CB.
  • New: control IupProgressBar.
  • New: APPENDNEWLINE and PADDING attribute for IupText. CUEBANNER and FILTER attributes for IupText in Windows.
  • New: INVERTED and TICKSPOS attributes in IupVal.
  • New: MASK attribute for IupText, IupMultiline, IupList and IupMatrix. The iupmask functions are now obsolete, autofill option and MATCH_CB callback are not supported anymore.
  • New: function IupGetDialogChild and common attribute NAME.
  • New: support for image and text simultaneous in IupButton.
  • New: support for mnemonics in IupLabel, IupButton and IupToggle.
  • New: text formatting for IupText using FORMATTING and ADDFORMATTAG attributes in Windows and GTK. New attribute OVERWRITE for IupText when using text formatting.
  • New: new ALL and NONE values for SELECTION attribute in IupText. New attribute SCROLLTO. New attributes SCROLLTOPOS, CARETPOS and SELECTIONPOS using 0 based character position. Function IupTextConvertXYToChar to convert (x,y) coordinates in (lin, col, pos) character positioning.
  • New: VALUEPOS and VALUE_HANDLE attributes for IupZbox.
  • New: PADDING, VALUE_HANDLE, VALUEPOS, MULTILINE and TABIMAGE attributes for IupTabs.
  • New: CLIPRECT attribute in IupCanvas.
  • New: TIPRECT auxiliar attribute for the TIP common attribute.
  • New: VIRTUALSCREEN and MONITORSINFO global attributes.
  • New: function IupGetChildPos and IupGetChildCount.
  • New: RADIO attribute for IupToggle.
  • New: AUTOTOGGLE and TITLEIMAGE attributes for IupItem. BGCOLOR support for IupMenu. Submenu now supports the IMAGE attribute. RADIO attribute for IupMenu.
  • New: SPIN, SPINVALUE, SPINMIN, SPINMAX, SPININC, SPINALIGN and SPINWRAP attributes in IupText. New SPIN_CB callback in IupText.
  • New: CANFOCUS attribute for IupButton, IupTogle, IupText, IupCanvas and IupVal.
  • New: VISIBLECOLUMNS, VISIBLELINES attributes for IupText.
  • New: APPENDVALUE, CANFOCUS, COUNT, DRAGDROP, INSERTITEMn, REMOVEITEM, TOPITEM, SPACING, VISIBLECOLUMNS, VISIBLELINES attributes for IupList.
  • New: BUTTON_CB, DBLCLICK_CB, DROPDOWN_CB, DROPFILES_CB, MOTION_CB callbacks for IupList.
  • New: IupListConvertXYToItem function for IupList.
  • New: IupMainLoopLevel function.
