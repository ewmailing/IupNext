IupLayoutDialog (since 3.3)

Creates a Layout Dialog. It is a predefined dialog to visually edit the layout of another dialog in run time. It is a standard IupDialog constructed with other IUP elements inside. The dialog can be shown with any of the show functions IupShow, IupShowXY or IupPopup.

The dialog is composed by two areas: one showing the given dialog children hierarchy tree, and one displaying its layout.

Creation

Ihandle* IupLayoutDialog(Ihandle* dialog); [in C]
iup.layoutdialog{dialog: ihandle} -> (elem: ihandle) [in Lua]
[NOT available in LED]
dialog: identifier of the dialog to display the layout.

Returns: the identifier of the created element, or NULL if an error occurs.

Attributes

BACKCOLOR: The background color of the area where the layout is drawn. Default: "255 255 255".

FORECOLOR: The color used to draw the lines that displays the layout of each element that has a native representation. Default: "0 0 0".

FORECOLORVOID: The color used to draw the lines that displays the layout of each element that does not have a native representation, like IupVbox and IupFill for example. Default: "100 100 100".

DESTROYWHENCLOSED: The dialog will be automatically destroyed when closed. Default: Yes.

SHOWHIDDEN: Flag to display hidden elements in the layout. Default: NO.

Check the IupDialog attributes.

Callbacks

Check the IupDialog callbacks.

Notes

The selected dialog does not need to be mapped on the system nor visible.

Menus

Dialog
	New  - creates a new empty dialog, it will be destroyed when the layout is destroyed.
	Load - loads an existent dialog from the application.
	Load Visible - lists only visible dialogs from the application to be loaded.
	Reload - reloads the current dialog into the Layout Dialog.
	Refresh - recalculates the dialog layout of the current dialog (IupRefresh).
	Redraw - send a redraw to the current dialog (IupRedraw).
	Show - shows the current dialog (IupShow) 
	Close - hides the Layout Dialog, optionally self destroy according to DESTROYWHENCLOSED.
Layout
	Hierarchy - shows or hides the layout hierarchy tree at left.
	Update - redraw the layout.
	Auto Update - periodically redraw the layout.
	Opacity - controls the Layout Dialog opacity so you can composite it on top of the selected dialog.

Use Reload when the dialog has been changed and the layout was modified by the application. Use Update when attributes of the dialog were changed by the application and the layout needs to be redrawn.

Display

The layout of an element is drawn with its Current size using its FONT, TITLE, BGCOLOR and FGCOLOR if any. But inheritance is not used intentionally to emphasize the element attributes. Only the first line (limited to 50 characters in the tree) is used from the element TITLE.

Examples

IupShow(IupLayoutDialog(NULL));
